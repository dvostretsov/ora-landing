4. Как мы обрабатываем данные

4.1 Обработка диалогов

Для того чтобы ORA могла отвечать вам:

  • Ваши сообщения передаются в Claude API (Anthropic, Inc., США) в расшифрованном виде. Это технически необходимо — нейросеть не может работать с зашифрованным текстом.
  • Мы, как оператор, имеем технический доступ к вашим диалогам через инфраструктуру backend-сервера. Мы не читаем диалоги в рутинном режиме и не используем их в коммерческих целях.
  • Сообщения хранятся в базе данных в зашифрованном виде (AES-256-GCM). Прямой доступ к базе данных без ключа шифрования не позволяет прочитать содержимое сообщений.

4.2 Аналитический журнал ORA

После примерно 10 сессий ORA автоматически генерирует аналитический журнал (краткий анализ тем, целей и динамики). Журнал доступен только администратору системы и консультанту. Пользователю журнал напрямую недоступен (функция планируется в будущих версиях).

4.3 Экспорт консультанту

По расписанию или вручную администратор формирует PDF-отчёт для консультанта. Сырые тексты диалогов в отчёт не включаются.

7. Срок хранения данных

  • Сообщения и цели: хранятся на протяжении действия подписки + 30 дней после её истечения.
  • Аккаунт: при удалении аккаунта все данные удаляются каскадно в течение 30 дней.
  • Аналитические журналы: хранятся до момента удаления аккаунта.
  • Логи сервера: автоматически удаляются через 30 дней (политика Railway).

8. Безопасность данных

  • Сообщения в базе данных зашифрованы алгоритмом AES-256-GCM (аутентифицированное шифрование). Ключ шифрования хранится отдельно от данных.
  • Передача данных между клиентом и сервером — только по HTTPS/TLS 1.2+.
  • Аутентификация — через Supabase Auth с JWT-токенами.
  • Доступ к Admin Panel ограничен: только зарегистрированные администраторы.

9. Ваши права (GDPR)

  • Доступа — получить копию ваших персональных данных.
  • Исправления — потребовать уточнения неточных данных.
  • Удаления («право на забвение») — удалить аккаунт и все данные.
  • Переносимости — получить данные в машиночитаемом формате.
  • Ограничения обработки — в случаях, предусмотренных GDPR.
  • Возражения — против обработки на основании законного интереса.
  • Отзыва согласия — в любой момент, если обработка основана на согласии.

Для реализации прав обращайтесь: privacy@ora-app.io. Ответ — в течение 30 дней.

Если вы считаете, что ваши права нарушены, вы вправе подать жалобу в надзорный орган по защите данных Испании — AEPD (Agencia Española de Protección de Datos): aepd.es.

10. Дети

ORA не предназначен для лиц моложе 18 лет. Мы не собираем намеренно данные несовершеннолетних. Если вам стало известно о таком случае, свяжитесь с нами по адресу privacy@ora-app.io.

11. Изменения политики

При существенных изменениях мы уведомим вас через экран согласия в приложении. Незначительные редакционные правки вступают в силу немедленно после публикации.

4. How We Process Data

4.1 Conversations

  • Your messages are transmitted to the Claude API (Anthropic, Inc., USA) in decrypted form. This is technically necessary — a neural network cannot process encrypted text.
  • As the operator, we have technical access to your conversations through the backend server infrastructure. We do not read conversations routinely and do not use them for commercial purposes.
  • Messages are stored in the database encrypted (AES-256-GCM). Direct database access without the encryption key does not allow reading message content.

4.2 ORA Analytical Journal

After approximately 10 sessions, ORA automatically generates an analytical journal. The journal is accessible only to the system administrator and consultant. Direct user access is planned for a future release.

4.3 Consultant Export

The administrator generates a PDF report for the consultant on a schedule or manually. Raw conversation texts are not included in the report.
